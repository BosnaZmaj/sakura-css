Accessibility Improvements

Changes:

  • Add comprehensive focus states (~120 lines CSS)
  • Blue focus ring (#5b8bf5, 2px) for keyboard navigation
  • Button focus: outline + shadow glow
  • Card focus: outline + shadow
  • Dropdown item focus: background + outline
  • Link focus: outline + subtle background
  • Note: Input focus states intentionally excluded (existing styles preserved)

Utilities Added:

  • .skip-to-main - Skip link for keyboard users
  • .sr-only - Screen reader only content
  • Support for @media (prefers-contrast: high)
  • Support for @media (prefers-reduced-motion: reduce)

Fixes:

  • Focus reset refined to *:focus:not(:focus-visible)
  • Safer pattern that preserves keyboard navigation
Impact: Full keyboard navigation support. WCAG 2.1 AA/AAA compliance. Better experience for users with disabilities.

Implementation Details:

sakura.css - Accessibility Features

Location: Lines 203-321

Utility Classes:

  • .skip-to-main - Hidden until keyboard focus, jumps to main content
  • .sr-only - Screen reader only text (visually hidden)

Focus States (Blue #5b8bf5):

  • Buttons: 2px outline + 4px shadow glow
  • Cards (envelope, goal, account, pricing): 2px outline + shadow
  • Dropdowns: 2px outline + background tint
  • Links: 2px outline + subtle background
  • Navigation items: 2px outline + background
  • Checkboxes/Radios: 2px outline
  • Inputs: Not modified (existing styles preserved per user request)

Focus Management:

  • *:focus:not(:focus-visible) - Removes mouse click outlines
  • *:focus-visible - Shows outlines only for keyboard navigation
  • Preserves accessibility while improving visual experience

High Contrast Support:

  • @media (prefers-contrast: high)
  • Increases outline width to 3px
  • Uses currentColor for better contrast
  • Removes shadows (can reduce contrast)

Reduced Motion Support:

  • @media (prefers-reduced-motion: reduce)
  • Disables all animations and transitions
  • Sets durations to 0.01ms
  • Removes scroll-behavior smooth
  • Respects user motion preferences

WCAG Compliance:

  • 2.4.1 Bypass Blocks (skip link)
  • 2.4.7 Focus Visible (keyboard navigation)
  • 1.4.11 Non-text Contrast (focus indicators)
  • 2.3.3 Animation from Interactions (reduced motion)
  • 1.4.6 Contrast Enhanced (high contrast support)
